## What it is

**free-for.dev** is a curated list of software (SaaS, PaaS, IaaS) and other offerings with
**free developer tiers** — specifically scoped to things infrastructure developers (System
Administrators, DevOps Practitioners) are likely to find useful. Hosted at
[free-for.dev](https://free-for.dev), source on
[GitHub (ripienaar/free-for-dev)](https://github.com/ripienaar/free-for-dev). ~129k stars,
1,600+ contributors, maintained via pull requests. No explicit license (awesome-list
convention; the repo metadata reports none).

## Best for

- **AI agents picking infrastructure/services for a project on a budget** — before committing
  to a paid provider, look up free-tier options by category (cloud, DB, DNS, email,
  monitoring, auth, CI/CD, storage, generative AI, and 50+ more).
- **DevOps architects comparing free-tier ceilings** — e.g. GCP App Engine's 28 frontend
  instance hours/day, Cloud Firestore's 50k reads/day, etc. The list cites the actual free
  limits per service.
- **Anyone scoping a side project or MVP** — find hosting, databases, email, auth, and more
  without spending.

## Not for

- Self-hosted software — the list is explicitly **as-a-Service offerings only**, not
  self-hostable tools. (Per the maintainer's note: "This list is only for as-a-Service
  offerings, not for self-hosted software.")
- Free trials — a service must offer a **free tier**, not just a free trial. Time-bucketed
  free tiers must be at least a year.
- Paid-only TLS — services that restrict TLS to paid tiers are excluded (security posture
  matters).

## How it works

The list is a single large markdown file (`README.md`) organized into 50+ category sections,
each listing services with their free-tier limits cited inline (e.g. "28 frontend instance
hours per day", "1GB storage, 50,000 reads/day"). The [free-for.dev](https://free-for.dev)
site is a GitHub Pages render of that README. Maintenance is community-driven via pull
requests, reviewed by the maintainer who enforces the eligibility rules (as-a-Service only,
real free tier, ≥1 year if time-bucketed, no paid-only TLS).

## Brief tutorial

Browse by category:

1. Open [free-for.dev](https://free-for.dev) (or the
   [GitHub README](https://github.com/ripienaar/free-for-dev)).
2. Use the Table of Contents to jump to the category you need (Cloud, DNS, Email, Auth,
   Monitoring, Generative AI, etc.).
3. Each entry names the service and cites its free-tier limit.
4. Click through to the service to sign up.

An AI agent workflow: when a project needs infrastructure (hosting, DB, email, auth, etc.),
consult free-for.dev for the relevant category to find free-tier options before recommending
a paid provider.
